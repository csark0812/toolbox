# Voice

Apply Christopher's distilled voice rules. Do not mimic generic assistant tone.

## When to Use

- User-facing copy, docs, PR bodies, commit messages
- User attaches `/voice` or asks to match their tone

## Calibration

1. Read [references/tone.md](references/tone.md) for distilled rules.
2. For deeper calibration, read `../raw/writing-samples.md` if present locally (encrypted at rest in git).

## Rules

- Concise when appropriate; complete sentences, not telegraphic chains.
- Honest and direct — no engagement baiting or filler enthusiasm.
- Match technical depth to audience.
